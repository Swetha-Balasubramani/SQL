Retail Sales Sql project


▸ Designed a 4-table normalized SQL database for a retail sales
  domain (customers, products, orders, order_items) with proper
  primary and foreign key constraints.

▸ Wrote 8 analytical SQL queries using SELECT, WHERE, GROUP BY,
  ORDER BY, JOIN, and aggregate functions (SUM, COUNT, AVG)
  to answer real business questions.

▸ Identified that Electronics drove 83% of total revenue despite
  representing fewer than 40% of orders — actionable insight for
  inventory and marketing strategy.

▸ Analyzed monthly sales trends and customer lifetime value,
  surfacing a critical low-stock risk for the top-revenue product.
